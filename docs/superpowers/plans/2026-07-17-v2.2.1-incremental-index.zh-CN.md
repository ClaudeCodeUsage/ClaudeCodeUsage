# V2.2.1 ClaudeUsageIndex 增量索引 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 用 extension 生命周期内的 per-file `ClaudeUsageIndex` 取代每次变化后重读全部 JSONL，使 unchanged 文件零重读，同时保证增量输出与 fresh cold load 深度相等。

**已批准设计：** [V2.2.1 稳定化设计（简体中文）](../specs/2026-07-17-v2.2.1-stabilization-design.zh-CN.md)

**Architecture:** 先在未改生产代码时冻结 V2.2.0 的 legacy golden 输出；然后把“单文件解析贡献”“跨文件全局聚合”“cold adapter”作为一个原子、始终可编译的重构提交，并用 golden 证明结果不变。随后让 `ClaudeUsageIndex` 以 path/size/mtime/dev/ino manifest 与外部 attribution dependency fingerprints 缓存单文件贡献及最近一次 aggregate；watcher 文件名只作安全 hint，startup/manual/每 30 分钟做权威 reconciliation。changed/new 文件在 V2.2.1 仍整文件重读，deleted/truncated/replaced 文件立即失效；基础 `refresh()` 与 single-flight coalescing 分两个提交。extension 在被动 trigger 且 `dataChanged=false` 时跳过 heavy usage breakdown/webview，但继续更新 quota/status；manual/settings/pricing/workspace 仍用 cached records 刷新 presentation。

**Tech Stack:** TypeScript 5.8、Node.js 标准库、`node:test`、VS Code Extension API；依赖前置计划 `2026-07-17-v2.2.1-refresh-mitigation.zh-CN.md` 提供的 manifest、refresh trigger 和匿名 diagnostics 接口。

## Global Constraints

- 目标分支为 `codex/v2.2.1-stabilization`，基线为已发布的 `v2.2.0`。
- 必须先完成并通过 `2026-07-17-v2.2.1-refresh-mitigation.zh-CN.md`；本计划直接消费其中的 `UsageFileFingerprint`、`UsageManifest`、`scanUsageManifest()`、`RefreshTrigger` 和 `LoadUsageDiagnostics`。
- V2.2.1 仍是 Claude-only；不得加入 Codex ingestion、provider UI 或跨设备同步。
- 不增加 runtime dependency。
- cache 只存在内存；不得写入磁盘、`globalState`、workspaceState 或日志。
- manifest fingerprint 精确包含 path、size、mtime、dev、ino（平台无 dev/ino 时回退 size+mtime）。
- unchanged 文件复用贡献；changed/new 文件整文件重读；deleted 文件移除贡献。
- truncation、inode change 与 atomic replacement 必须失效；V2.2.1 不实现 append byte-offset/tail parser。
- watcher filename 只是 hint；startup、manual 与每 30 分钟 reconciliation 以完整 manifest 为权威。
- parsing contract、attribution rule、`analyzeContent` 和 content-analysis window 必须进入 cache key。
- `windowDays` 在 request 边界统一规范化为 `Math.min(365, Math.max(1, Math.round(value)))`；contribution key、aggregate key、cutoff 和 equality test 必须消费同一个规范化值。
- 全局 placeholder/real 去重、session title、用户消息计数、workflow/subagent attribution 与 content calibration 的结果不得变化。
- 文件顺序必须复刻 V2.2.0：按每个文件“首个合法 timestamp”（不是最小 timestamp）做稳定排序；相同/无效 timestamp 保留 manifest discovery order，禁止增加 path/`localeCompare()` tie-break。
- `agent-*.meta.json` 与 `<session>/workflows/scripts` 是 JSONL 的外部 attribution dependency；其 fingerprints 变化必须让输出失效，workflow-name cache key 必须包含 `scriptsDir + workflowId`。
- unchanged JSONL、unchanged dependencies 且 analysis 未到下一条 expiry 时必须复用最后一次 source aggregate，返回 `dataChanged=false`。被动的 poll/watch/focus/credentials 触发不得重跑 usage breakdown 或 webview render；quota/status 仍可用 cached usage 更新，manual/settings/pricing/workspace 等显式触发仍要刷新 presentation。
- index 更新必须事务化：读/解析失败、hint 非 `ENOENT` 错误或 read-after-stat race 都不得部分提交 `entries`、manifest 或 aggregate。
- cached contribution 与 `lastAggregate` 视为不可变；diagnostics 包装不得改写 cached records/contentAnalysis，下游只读消费。测试在 strict fixture 中 deep-freeze cached result 后执行一次 unchanged refresh。
- JSONL read 必须在 read 后 `fstat()` 验证 identity/size/mtime；发生竞态只重试一次，第二次仍变化则本轮失败并保留旧 snapshot。
- hint 必须 containment-check 到 resolved `<root>/projects`，用 `lstat()` 拒绝 symlink；`ENOENT` 只表示删除，其他错误 fail closed。
- cache 的确定性内存上界为“当前 manifest 的每个 JSONL 一份 compact contribution + 当前外部 dependency fingerprint + 一份 aggregate”；不得缓存 raw JSONL/raw parsed object，删除/replace/`clear()` 后不得保留旧 contribution 的强引用。analysis 只保留 parser 当时 window cutoff 之后的 compact events；每文件 prompt candidates 最多保留 legacy 所需的最后 600 条。skill/command candidates 必须保留该 coverage window 内的全部 compact 候选，legacy first-5000 cap 只在全局 replay/finalize 应用，否则 cutoff 前进后第 5001+ 项无法补位。analysis line 对象数不得超过该文件 `linesParsed`，嵌套候选数与实际 parsed content blocks 线性对应。
- 诊断不得包含 prompt、绝对路径、session ID、credential 或原始 JSONL 行。
- 现有 loader 语义诊断与 `log` 输出必须保留：`rejected` reason、dedup replace/skip、kept、user prompts 与 model count/token summaries 不能被性能诊断取代。
- 不纳入 PR #73 的 1000-session 渲染；不借此重构 webview。
- 每项行为变化先 RED、再 GREEN；`npm test` 必须零失败。
- 若 cold-versus-incremental deep equality 未通过，则 V2.2.1 只交付前置 mitigation，保持 #70 开放，不合入本索引。

- [ ] **Step 3: 建立精确贡献类型，并把现有逐行分支机械迁入 parser**

```ts
// src/claudeUsageContribution.ts — public contract
import { ClaudeUsageRecord } from './types';
import { UsageFileFingerprint } from './claudeUsageFiles';

export const PARSER_CONTRACT_VERSION = 1;
export const ATTRIBUTION_CONTRACT_VERSION = 1;

export function normalizeClaudeUsageRequest<T extends { windowDays: number }>(request: T): T {
  return {
    ...request,
    windowDays: Math.min(365, Math.max(1, Math.round(request.windowDays))),
  };
}

export interface ContributionCacheKey {
  parserVersion: number;
  attributionVersion: number;
  analyzeContent: boolean;
  windowDays: number;
}

export interface RecordCandidate {
  record: ClaudeUsageRecord;
  uniqueHash: string | null;
  line: number;
}

export interface TitleCandidate {
  sessionId: string;
  kind: 'ai' | 'custom';
  value: string;
  line: number;
}

export interface AnalysisDelta {
  key: string;
  estimatedTokens: number;
  charCount: number;
  count: number;
}

export interface NormalizedAnalysisLine {
  timestamp: string | null;
  uuid: string | null;
  sessionId: string;
  categoryDeltas: AnalysisDelta[];
  toolDeltas: AnalysisDelta[];
  toolUses: { id: string; name: string; skill?: string }[];
  toolResults: {
    toolUseId: string;
    estimatedTokens: number;
    charCount: number;
    count: number;
  }[];
  prompts: { cwd: string; text: string }[];
  thinking?: { tokens: number; total: number; hidden: boolean };
  commandUses: { name: string; estimatedTokens: number }[];
}

export interface ClaudeFileContribution {
  fingerprint: UsageFileFingerprint;
  cacheKey: ContributionCacheKey;
  // V2.2.0 sortFilesByTimestamp() uses the first valid timestamp, not min().
  // 0 means no valid timestamp and therefore sorts first; stable input order
  // resolves ties.
  firstValidTimestampMs: number;
  /** Earliest timestamp retained for content-analysis replay. */
  analysisCoveredFromMs: number;
  recordCandidates: RecordCandidate[];
  titles: TitleCandidate[];
  analysisLines: NormalizedAnalysisLine[];
  bytesRead: number;
  linesParsed: number;
  parseErrors: number;
  rejected: Record<string, number>;
  attributionDependencies: AttributionDependencyRef[];
}

export interface AttributionDependencyRef {
  kind: 'agent-meta' | 'workflow-scripts';
  path: string;
  workflowId?: string;
}

export interface AttributionDependencyFingerprint extends AttributionDependencyRef {
  exists: boolean;
  size: number;
  mtimeMs: number;
  dev: number;
  ino: number;
  directorySignature?: string;
}

export interface ParseClaudeUsageOptions {
  analyzeContent: boolean;
  windowDays: number;
  nowMs: number;
}

export function parseClaudeUsageContent(
  fingerprint: UsageFileFingerprint,
  content: string,
  options: ParseClaudeUsageOptions
): ClaudeFileContribution {
  const normalized = normalizeClaudeUsageRequest(options);
  const lines = content.split('\n');
  const cacheKey: ContributionCacheKey = {
    parserVersion: PARSER_CONTRACT_VERSION,
    attributionVersion: ATTRIBUTION_CONTRACT_VERSION,
    analyzeContent: normalized.analyzeContent,
    windowDays: normalized.windowDays,
  };
  const contribution: ClaudeFileContribution = {
    fingerprint,
    cacheKey,
    firstValidTimestampMs: 0,
    analysisCoveredFromMs: normalized.nowMs - cacheKey.windowDays * 86_400_000,
    recordCandidates: [],
    titles: [],
    analysisLines: [],
    bytesRead: Buffer.byteLength(content, 'utf8'),
    linesParsed: 0,
    parseErrors: 0,
    rejected: {},
    attributionDependencies: deriveAttributionDependencyPaths(fingerprint.path),
  };

  // 将 dataLoader 当前每个 file 内的 sessionInfo、isSubagentFile、agentInfo、
  // recentPrompts 初始化移到这里。下面循环严格保留当前分支顺序：analysis →
  // title → agent task → genuine user prompt → validate → attribution tags。
  for (let line = 0; line < lines.length; line += 1) {
    if (lines[line].trim() === '') continue;
    contribution.linesParsed += 1;
    let parsed: unknown;
    try { parsed = JSON.parse(lines[line]); } catch {
      contribution.parseErrors += 1;
      continue;
    }
    // 遇到第一个可解析且 timestamp 合法的对象时只赋值一次
    // firstValidTimestampMs；不得取整文件 min(timestamp)。
    // normalizeAnalysisLine 只保存 analysisCoveredFromMs 之后的上方 compact
    // deltas，不保存 raw line；usage record candidates 仍覆盖完整历史。
    // 同一 user line 的每个 text block 各追加到 prompts[]，不能覆盖成单个 prompt。
    // tool_use 与 tool_result 只保存 id/name/skill 和 token/chars/count；映射留给
    // Task 2 的 global replay，使 tool id 可以跨文件关联。
    // title/candidate 的具体字段赋值逐字沿用 dataLoader 当前实现。
  }
  // 保留每文件最后 600 个 prompt candidates；保留 coverage window 内全部
  // compact skill/command candidates。first-5000 只能在 global replay/finalize
  // 应用，这样 cutoff 前进后原第 5001+ 个候选可以正确补位。
```

```ts
export interface AggregateOptions {
  analyzeContent: boolean;
  windowDays: number;
  nowMs: number;
}

export interface AggregatedClaudeUsage {
  records: ClaudeUsageRecord[];
  contentAnalysis: ContentAnalysis | null;
  diagnostics: LoadUsageDiagnostics;

  dataRoots: readonly string[];

function tokenSum(record: ClaudeUsageRecord): number {
  const usage = record.message.usage;
  return (usage.input_tokens || 0) + (usage.output_tokens || 0) +
    (usage.cache_creation_input_tokens || 0) + (usage.cache_read_input_tokens || 0);
}

export function aggregateClaudeContributions(
  input: readonly ClaudeFileContribution[],
  options: AggregateOptions
): AggregatedClaudeUsage {
  const normalized = normalizeClaudeUsageRequest(options);
  // Array.prototype.sort is stable in the supported Node runtime. Do not add
  // a path/localeCompare tie-break: V2.2.0 preserves discovery order on ties.
  const contributions = [...input].sort(
    (a, b) => a.firstValidTimestampMs - b.firstValidTimestampMs
  );
  const records: ClaudeUsageRecord[] = [];
  const processedHashes = new Map<string, number>();
  const aiTitles = new Map<string, string>();
  const customTitles = new Map<string, string>();
  const analysis = normalized.analyzeContent
    ? createGlobalAnalysisAccumulator(normalized.nowMs - normalized.windowDays * 86_400_000)
    : null;

  for (const contribution of contributions) {
    for (const title of [...contribution.titles].sort((a, b) => a.line - b.line)) {
      (title.kind === 'custom' ? customTitles : aiTitles).set(title.sessionId, title.value);
    }
    for (const candidate of [...contribution.recordCandidates].sort((a, b) => a.line - b.line)) {
      // Cached contributions are immutable; title attachment happens on a copy.
      const record = { ...candidate.record };
      if (candidate.uniqueHash && processedHashes.has(candidate.uniqueHash)) {
        const oldIndex = processedHashes.get(candidate.uniqueHash)!;
        if (tokenSum(record) > tokenSum(records[oldIndex])) {
          records[oldIndex] = record;
        }
      } else {
        records.push(record);
        if (candidate.uniqueHash) processedHashes.set(candidate.uniqueHash, records.length - 1);
      }
    }
    if (analysis) replayNormalizedAnalysis(contribution.analysisLines, analysis);
  }

  for (const record of records) {
    if (!record._sessionId) continue;
    const title = customTitles.get(record._sessionId) || aiTitles.get(record._sessionId);
    if (title) record._sessionTitle = title;
  }
  const contentAnalysis = analysis ? finalizeGlobalAnalysis(analysis, records) : null;
  return {
    records,
    contentAnalysis,
    diagnostics: {
      filesDiscovered: contributions.length,
      filesChanged: contributions.length,
      filesReused: 0,
      filesRemoved: 0,
      filesFailed: 0,
      bytesRead: contributions.reduce((sum, item) => sum + item.bytesRead, 0),
      linesParsed: contributions.reduce((sum, item) => sum + item.linesParsed, 0),
      manifestMs: 0,
      readParseMs: 0,
    },
  };
}

export async function loadClaudeUsageCold(
  request: ColdLoadRequest
): Promise<AggregatedClaudeUsage> {
  const normalized = normalizeClaudeUsageRequest(request);
  const manifestStarted = performance.now();
  const manifest = await scanUsageManifest(normalized.dataRoots);
  const manifestMs = performance.now() - manifestStarted;
  const readParseStarted = performance.now();
  const contributions = [] as ClaudeFileContribution[];
  for (const fingerprint of manifest.entries.values()) {
    const content = await readFile(fingerprint.path, 'utf8');
    contributions.push(parseClaudeUsageContent(fingerprint, content, normalized));
  }
  const result = aggregateClaudeContributions(contributions, normalized);
  result.diagnostics.manifestMs = manifestMs;
  result.diagnostics.readParseMs = performance.now() - readParseStarted;
  return result;
}
```

```ts
    a.attributionVersion === b.attributionVersion &&
    a.analyzeContent === b.analyzeContent && a.windowDays === b.windowDays;

  private lastAggregateKey = '';
  private lastManifestOrderKey = '';

  async refresh(request: ClaudeUsageRefreshRequest): Promise<IndexedClaudeUsage> {

  private async runOnce(request: ClaudeUsageRefreshRequest): Promise<IndexedClaudeUsage> {
    const rootsKey = request.dataRoots.map((root) => path.resolve(root)).join('\0');
    const rootsChanged = this.resolvedRootsKey !== '' && rootsKey !== this.resolvedRootsKey;
    const manifestStarted = performance.now();
    const { manifest, usedFullScan } = await buildManifestForRequest(
      rootsChanged ? null : this.lastManifest,
      request,
      this.lastFullReconciliationAt,
      this.hooks
    );
    const manifestMs = performance.now() - manifestStarted;
    const delta = diffUsageManifests(rootsChanged ? null : this.lastManifest, manifest);
    const manifestOrderKey = [...manifest.entries.values()]
      .map((entry) => `${entry.discoveryIndex}\0${entry.path}`)
      .join('\0');
    const manifestOrderChanged = manifestOrderKey !== this.lastManifestOrderKey;
    const readParseStarted = performance.now();
    const cacheKey: ContributionCacheKey = {
      parserVersion: PARSER_CONTRACT_VERSION,
      attributionVersion: ATTRIBUTION_CONTRACT_VERSION,
      analyzeContent: request.analyzeContent,
      windowDays: request.windowDays,
    };
    let bytesRead = 0;
    let linesParsed = 0;
    let changed = 0;
    let reused = 0;
    const requestedAnalysisCutoff = request.nowMs - cacheKey.windowDays * 86_400_000;
    // Transaction: nextEntries shares immutable old contributions, but no
    // live field changes until every changed file/dependency is stable.
    const nextEntries = new Map(rootsChanged ? [] : this.entries);
    for (const removed of delta.removed) nextEntries.delete(removed.path);
    for (const fingerprint of manifest.entries.values()) {
      const cached = this.entries.get(fingerprint.path);
      const dependencies = await fingerprintAttributionDependencies(fingerprint.path);
      if (cached && sameFingerprint(cached.fingerprint, fingerprint) &&
          sameKey(cached.cacheKey, cacheKey) && sameDependencies(cached.dependencies, dependencies) &&
          (!request.analyzeContent ||
           requestedAnalysisCutoff >= cached.contribution.analysisCoveredFromMs)) {
        reused += 1;
        continue;
      }
      this.hooks.onFileRead?.(fingerprint.path);
      const stable = await readStableContribution(
        fingerprint, cacheKey, request.nowMs, dependencies, this.hooks
      );
      nextEntries.set(fingerprint.path, {
        fingerprint,
        cacheKey,
        contribution: stable.contribution,
        dependencies: stable.dependencies,
      });
      bytesRead += stable.contribution.bytesRead;
      linesParsed += stable.contribution.linesParsed;
      changed += 1;
    }
    const aggregateKey = `${rootsKey}\0${cacheKey.analyzeContent}\0${cacheKey.windowDays}`;
    const hasSemanticDelta = changed > 0 || delta.removed.length > 0 || manifestOrderChanged ||
      aggregateKey !== this.lastAggregateKey || request.nowMs >= this.nextAnalysisExpiryAt;
    if (!hasSemanticDelta && this.lastAggregate) {
      this.entries = nextEntries;
      this.lastManifest = manifest;
      this.lastManifestOrderKey = manifestOrderKey;
      this.resolvedRootsKey = rootsKey;
      if (usedFullScan) this.lastFullReconciliationAt = request.nowMs;
      return withRefreshDiagnostics(this.lastAggregate, {
        filesDiscovered: manifest.entries.size, filesChanged: 0, filesReused: reused,
        filesRemoved: 0, filesFailed: 0, bytesRead: 0, linesParsed: 0, manifestMs,
        readParseMs: performance.now() - readParseStarted,
      }, false);
    }
    this.hooks.onAggregate?.();
    const aggregated = aggregateClaudeContributions(
      [...manifest.entries.keys()].map((filePath) => nextEntries.get(filePath)!.contribution),
      { analyzeContent: request.analyzeContent, windowDays: cacheKey.windowDays, nowMs: request.nowMs }
    ) as IndexedClaudeUsage;
    aggregated.diagnostics = {
      ...aggregated.diagnostics,
      filesDiscovered: manifest.entries.size,
      filesChanged: changed,
      filesReused: reused,
      filesRemoved: delta.removed.length,
      filesFailed: 0,
      bytesRead,
      linesParsed,
      manifestMs,
      readParseMs: performance.now() - readParseStarted,
    };
    aggregated.dataChanged = true;
    // Commit all state only after parsing + aggregate succeeds.
    this.entries = nextEntries;
    this.lastManifest = manifest;
    this.lastManifestOrderKey = manifestOrderKey;
    this.lastAggregate = aggregated;
    this.lastAggregateKey = aggregateKey;
    this.nextAnalysisExpiryAt = computeNextAnalysisExpiry(nextEntries, request.nowMs, cacheKey.windowDays);
    this.resolvedRootsKey = rootsKey;
    if (usedFullScan) this.lastFullReconciliationAt = request.nowMs;
    return aggregated;
  }
}
```

`withRefreshDiagnostics()` 必须返回新的顶层 result 和新的 diagnostics 对象（`{ ...cached, diagnostics: { ...metrics }, dataChanged: false }`），不得给 `lastAggregate` 或其 records/contentAnalysis 赋值；这些 payload 由下游只读复用。

`readStableContribution()` 返回 `{ contribution, dependencies: stableDependencies }`，caller 只能缓存这个返回的 dependency snapshot，不能缓存调用前的 tentative snapshot。每次 attempt 必须 `open(path, 'r')`，先 `handle.stat()`，再 `handle.readFile({ encoding: 'utf8' })`，调用 JSONL test hook 后再 `handle.stat()`；pre/post 与 manifest fingerprint 任一不相同就关闭 handle 并重试。随后以 attempt-local、全新的 `{ agentType, workflowName }` resolver caches 解析 attribution，调用 dependency-race test hook，再 fingerprint dependencies；若 pre/post dependency snapshot 不同，丢弃 contribution 和该 attempt 的 resolver caches，用新 snapshot 从 parse/resolve 完整重试一次。第二次 JSONL 或 dependency 仍变化时抛出匿名 `UsageFileChangedDuringReadError`，整轮 transaction 回滚。`finally` 总是 close；raw string 只作为 `parseClaudeUsageContent()` 的局部参数，返回前释放，不能写入 contribution/hook。parser 使用 request 的固定 `nowMs` 写入 `analysisCoveredFromMs`；向前推进的 cutoff 可复用已有 superset，窗口扩大或时钟回拨到 coverage 之前则整文件重读，绝不能在缺少历史 analysis events 时假装 cache hit。

`fingerprintAttributionDependencies()` 对 `agent-*.meta.json` 保存 file fingerprint；对 workflow 保存 `scriptsDir` 与“匹配当前 workflow id 的 script filename + 每个匹配文件 fingerprint”排序后的 `directorySignature`。resolver cache key 精确为 `scriptsDir + '\0' + workflowId`。meta/script 缺失使用 `exists=false` fingerprint，因此后续创建也能使 cache 失效。对 changed entry 要在 JSONL parse/attribution resolve 后再 fingerprint dependencies 一次；前后不同则与 JSONL race 使用同一“只重试一次、仍变化则整轮回滚”规则。retry 必须用新的 resolver caches，最终把 stable post snapshot 与对应 contribution 一起返回，避免“旧 attribution + 新 fingerprint”。cold adapter 使用同一稳定 dependency-attribution helper，而不是较弱的单次读取路径。

```ts
private inFlight: Promise<IndexedClaudeUsage> | null = null;
private pending: ClaudeUsageRefreshRequest | null = null;
private lastFullReconciliationAt = 0;
private static readonly RECONCILE_MS = 30 * 60 * 1000;

async refresh(request: ClaudeUsageRefreshRequest): Promise<IndexedClaudeUsage> {
  if (this.inFlight) {
    this.pending = this.pending ? mergeRequests(this.pending, request) : request;
    return this.inFlight;
  }
  this.inFlight = this.runLoop(request);
  try { return await this.inFlight; } finally {
    this.inFlight = null;
    this.pending = null;
  }
}

private async runLoop(initial: ClaudeUsageRefreshRequest): Promise<IndexedClaudeUsage> {
  let request: ClaudeUsageRefreshRequest | null = initial;
  let result!: IndexedClaudeUsage;
  while (request) {
    result = await this.runOnce(request);
    request = this.pending;
    this.pending = null;
  }
  return result;
}
```

`mergeRequests()` 使用 `mergeRefreshTrigger()`，`forceReconcile` 取 OR，`hintedPaths` 用 `Set` union，最新 request 的 `analyzeContent`、规范化 `windowDays`、`nowMs` 与 resolved `dataRoots` 覆盖旧值。若 pending request 的 resolved roots 与 in-flight 不同，pending 保留新 roots 并强制 reconciliation；绝不把两个 root 的 hints 混用。

所有等待同一个 in-flight 的 caller 都等待 `runLoop()` 完成并收到最后一次 follow-up result；index 在 finally 前执行合并后的唯一 follow-up，并把 `lastAggregate` 留在 follow-up 的结果。follow-up 失败时 `refresh()` reject，Task 3 的 transaction 保证首轮已提交 snapshot 不被半写；`pending` 在成功/失败路径都清空。这里不得重复实现 Task 3 的 manifest/hint/reconciliation 逻辑。

```ts
  options: UsageRefreshOptions = {}
```

## 完成定义

- cold start 每个 JSONL 恰好整文件读取一次；首 timestamp 不造成第二次全文件读取。
- unchanged refresh 的 JSONL `bytesRead=0`、`linesParsed=0`。
- unchanged passive `poll|watch|focus|credentials` refresh 的 parser read、source aggregate 与 extension usage breakdown/webview 调用次数都为 0，返回 `dataChanged=false`；quota/status 仍可更新，manual/settings/pricing/workspace 仍刷新 presentation。
- 一个 changed/new 文件只整文件读取该文件；V2.2.1 没有 append-offset 状态。
- delete/truncate/inode change/atomic replace 都移除旧贡献并重算 totals。
- watcher burst 只形成现有 single-flight 后的一个 pending follow-up；hint 丢失最多 30 分钟后由 reconciliation 恢复。
- analysis 开关/window、parser version、attribution version 改变会明确失效。
- `agent-*.meta.json` / workflow scripts 的 create/edit/delete 会使 attribution 失效；同 workflow id 的不同 `scriptsDir` 不串 cache。
- placeholder/real dedup、late custom title、workflow/subagent attribution、message count、content analysis 和 calibration 与 fresh cold load `deepStrictEqual`。
- cache 仅内存存在并满足 entry/raw/GC bound；性能诊断只含匿名数字，同时旧 loader rejection/dedup/model log 完整保留。
- data roots 只比较 canonical resolved roots；hint 越界/symlink/non-ENOENT 错误 fail closed，任何失败不产生部分 snapshot。
- 未触及 Codex provider、跨设备同步、append tail cache 或 1000-session UI。
- 完整 test/package/prerelease gate 全部通过；否则不合入 index，V2.2.1 只发布低风险 mitigation。
