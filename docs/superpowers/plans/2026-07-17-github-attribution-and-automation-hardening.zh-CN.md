# GitHub 归因与自动化加固 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 issue/PR first-pass 自动回复按“实际产生最终正文的 tier/provider”确定性添加且只添加一个可信 footer；同时把环境、PR diff、仓库文件读取、symlink、secret path 与 GitHub POST payload 收紧为 fail closed，并保持高权限 mention agent 的权限边界不变。

**已批准设计：** [V2.2.1 稳定化设计（简体中文）](../specs/2026-07-17-v2.2.1-stabilization-design.zh-CN.md)

**Architecture:** 把 provider/transport 校验、footer 包装、环境 preflight 和安全 repo reader 提取到 `.github/scripts/first-pass-lib.mjs`，用 Node 原生测试覆盖。`.github/scripts/first-pass.mjs` 仍使用现有 Anthropic Messages-compatible transport，只负责两级模型编排和 GitHub comment I/O；它分别校验 cheap/pro provider，并把最终被采用回复对应的 provider 传给 trusted wrapper。当前 transport 只允许 DeepSeek 或 Claude 自动归因；Codex 自动归因只有在未来真正实现并锁定 OpenAI/Codex transport 后才能启用。两个 comment-only workflow 只读 base repository；`.github/workflows/claude.yml` 完全不修改。

**Tech Stack:** Node.js 20 ESM、`node:test`、GitHub Actions YAML、现有 Anthropic Messages-compatible HTTP endpoint、GitHub Issues Comments REST API。

## Global Constraints

- Attribution 描述的是实际生成最终正文的 provider，不是 workflow 名称、客户端或曾经尝试过但未采用的 tier。
- cheap 与 pro 必须分别配置、分别校验：`CCU_BOT_GENERATOR` 对应 `CCU_BOT_MODEL`，`CCU_BOT_GENERATOR_PRO` 对应 `CCU_BOT_MODEL_PRO`。
- 当前 runner 的 transport 是代码内固定的 `anthropic-messages`，不是可由仓库变量伪造的字符串；它只允许 `deepseek` 和 `claude`。任何 `codex` 配置都必须在发请求或发 comment 前失败。
- 未来只有实现、测试并锁定真实的 `openai-responses-codex` transport 后，automatic Codex footer 才可启用；本计划仅为 pure policy 写一条 future-contract test，不实现该 transport。
- 维护者审阅过的 Codex 文本使用 `Generated with`；自动 first pass 使用 `Generated by ... as an automated first pass — not a maintainer decision.`。二者不可混用。
- trusted wrapper 而不是模型 prompt 添加 footer；模型输出中的 header/footer/disclaimer 即使藏在 blockquote、list、code fence、不同大小写或不同链接写法中也要剥离，最终 comment 只能有一个精确 suffix。
- first-pass workflow 只读 base repository、永不 checkout 或执行 PR head、只创建一条 comment，不 label/close/approve/merge/push。
- PR diff 获取失败或为空时停止，不发布无 diff 支撑的 review。
- 每次 runner 建立一个共享 repo-read session；grounding docs 与模型后续请求共同使用同一去重集合和预算。整个 comment 生命周期最多读取 6 个唯一文件、每文件 16,000 bytes、总计 60,000 bytes；预算按 UTF-8 bytes 而不是 JavaScript 字符数计算。
- readable path 是 strict allowlist：仅显式 root docs/config 与 `src/**`；同时应用 secret denylist。拒绝 hidden component、traversal、absolute path、NUL、symlink、root escape 与非文本扩展名。
- 公共 issue/PR 输入不能宣称为“prompt-injection safe”；准确表述为固定 runner 限制了模型能力，但模型输出仍可能错误。
- v2.2.1 不新增 runtime dependency；自动化测试只使用 Node 20 内置模块。
- `.github/workflows/claude.yml` 是独立的 maintainer-only 高权限 workflow；不得改 action、权限、trigger 或 allowlist，也不在 v2.2.1 迁移到 Codex。
- 本中文版计划是本轮 `CONTRIBUTING.md` 英文变更的中文审阅伴随说明。以后任何需要维护者审阅的 policy/contributor 文档，都应同时提供中文 sibling 或等价的中文 review companion。
- README tool credit 由 `2026-07-17-codex-takeover-hygiene.zh-CN.md` 负责；本计划不重复修改六份 README。
- `CHANGELOG.md` 已由前置 hygiene 计划建立 `## [2.2.1] — Unreleased`；本计划只能向该节追加，不得再建同名版本节。
- 每项行为变化遵循 red-green TDD；提交前所有 TypeScript 与 automation ESM tests 必须零失败。

- [ ] **Step 3: 实现 generator registry 与 transport gate**

Create `.github/scripts/first-pass-lib.mjs`. The registry and resolver must have this shape:

```js
const GENERATORS = Object.freeze({
  deepseek: Object.freeze({
    id: 'deepseek',
    label: 'DeepSeek',
    url: 'https://www.deepseek.com/',
    modelPattern: /deepseek/i,
    transports: new Set(['anthropic-messages']),
  }),
  claude: Object.freeze({
    id: 'claude',
    label: 'Claude',
    url: 'https://www.anthropic.com/claude',
    modelPattern: /(?:^|[-_/])claude(?:$|[-_/])/i,
    transports: new Set(['anthropic-messages']),
  }),
  codex: Object.freeze({
    id: 'codex',
    label: 'OpenAI Codex',
    url: 'https://developers.openai.com/codex/',
    modelPattern: /(?:^|[-_/])codex(?:$|[-_/])/i,
    transports: new Set(['openai-responses-codex']),
  }),
});

function inferGenerator(modelName) {
  const matches = Object.values(GENERATORS).filter((item) => item.modelPattern.test(modelName));
  if (matches.length !== 1) {
    throw new Error(`Cannot infer generator from model ${modelName}`);
  }
  return matches[0].id;
}

export function resolveGeneratorAttribution(rawId, rawModelName, transport) {
  const modelName = String(rawModelName || '').trim();
  if (!modelName) throw new Error('A model id is required for generator attribution');
  const id = String(rawId || inferGenerator(modelName)).trim().toLowerCase();
  const generator = GENERATORS[id];
  if (!generator) throw new Error(`Unsupported generator: ${id}`);
  if (!generator.modelPattern.test(modelName)) {
    throw new Error(`Model ${modelName} does not match configured generator ${id}`);
  }
  if (!generator.transports.has(transport)) {
    const required = [...generator.transports].join(' or ');
    throw new Error(`Generator ${id} requires transport ${required}; received ${transport}`);
  }
  return Object.freeze({
    id: generator.id,
    label: generator.label,
    url: generator.url,
    transport,
    modelName,
    automaticFooter:
      `🤖 Generated by [${generator.label}](${generator.url}) ` +
      'as an automated first pass — not a maintainer decision.',
  });
}
```

Important: runner 不能从 env 接受 `openai-responses-codex`；future-contract test 只证明 pure policy 的最低门槛，不代表 transport 已实现。

- [ ] **Step 4: 实现 stable-dedup、symlink-safe、UTF-8 byte budgets**

Implementation requirements:

1. Normalize and validate all requested paths.
2. `createRepoReadSession()` owns `seenPaths`、`filesRead` 与 `totalBytes`。每次 `read()` 都按 insertion order 规范化并查同一 `seenPaths`；已见路径不重复返回、不消耗 slot/bytes。hard maxima 在 session 生命周期内累计，不能由第二次调用重置。
3. Resolve the real repo root and verify lexical containment. For every candidate, split the normalized relative path and `lstat` every prefix from repo root through the leaf; reject if **any** component is a symlink, including a parent symlink that points back inside the same repository. Require the leaf to be a regular file, then `realpath` and verify real containment. On the Ubuntu Actions runtime, open with `O_RDONLY | O_NOFOLLOW`, `fstat` the descriptor, and verify the opened descriptor still corresponds to the validated real file before reading, so component replacement between check/open also fails closed.
4. Read at most `min(fileBudget, remainingBudget)` bytes from that descriptor. Never call `readFileSync(..., 'utf8').slice(...)`, because that counts UTF-16 code units and can load an unbounded file.
5. Backtrack a truncated buffer to the last complete UTF-8 code point before decoding. Report the bytes actually retained.
6. Catch missing/unreadable/replaced files and skip them; never include error details containing absolute paths or file contents.
7. Each `read()` returns only newly admitted files as `{ text, files: [{ path, text, bytes }], totalBytes }`; `totalBytes` is the session cumulative total. `snapshot()` returns cumulative `{ filesRead, totalBytes }` without content. Runner uses `.text`, tests use metadata.

Keep `maxFiles=6`, `fileBudget=16_000`, `totalBudget=60_000` as hard maxima even if caller supplies larger options; tests may only lower them.

- [ ] **Step 6: 修复 null-aware reply parsing，并记录实际 final tier/provider**

先把现有 `tag()` / `parse()` 移到 `first-pass-lib.mjs` 并导出。`tag()` 找不到标签时返回 `null`，找到空标签时返回空字符串；reply fallback 必须检查 null，不能使用 truthiness：

```js
export function parseFirstPassResponse(raw) {
  const taggedReply = tag(raw, 'reply');
  const fallback = raw.replace(/<control>[\s\S]*?<\/control>/i, '').trim();
  const reply = (taggedReply !== null ? taggedReply : fallback).trim();
  let control = { answerable: true, want_files: [] };
  const taggedControl = tag(raw, 'control');
  if (taggedControl) {
    try {
      const value = JSON.parse(taggedControl);
      control = {
        answerable: value.answerable !== false,
        want_files: Array.isArray(value.want_files) ? value.want_files : [],
      };
    } catch {
      // Fail closed on escalation: malformed control never grants more files.
    }
  }
  return { reply, ...control };
}

export function chooseFinalReply(cheap, pro) {
  if (pro && typeof pro.reply === 'string' && pro.reply.trim() !== '') {
    return { reply: pro.reply.trim(), generator: pro.generator };
  }
  if (cheap && typeof cheap.reply === 'string' && cheap.reply.trim() !== '') {
    return { reply: cheap.reply.trim(), generator: cheap.generator };
  }
  throw new Error('No non-empty model reply');
}
```
