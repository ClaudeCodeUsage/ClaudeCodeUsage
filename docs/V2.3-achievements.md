# V2.3 — Achievements (成就系统) — SPEC ONLY

> This is a **specification**. Do **not** implement achievement-system code in
> V2.2. V2.3 builds it on top of the V2.2 share-card + heatmap foundations.

## 1. Product positioning

- **Optional, local-first, fun but not intrusive.** A side feature, clearly
  separate from the core token dashboard.
- **Disabled by default, or local-only by default.** It must never affect
  performance or the primary UX.
- It is a delight layer for people who enjoy it, invisible to those who don't.

## 2. Privacy principles

- No prompt text leaves the machine. No file paths. No session ids. No raw local
  username. No absolute directories.
- All evaluation runs **locally** on the same aggregate data the dashboard
  already reads.
- Any public / community feature is strictly **opt-in**; public data is
  aggregated and anonymous.
- Do **not** use prompt *semantic* classification unless it is clearly opt-in and
  runs locally.

## 3. Achievement categories

Concrete, playful, internet-native ideas (names are the product copy; meanings
guide the local rule). All thresholds are deterministic and computed locally.

**Token / usage**
- `Token 小试牛刀` — first meaningful day of usage.
- `百万上下文居民` — crossed 1M cumulative context tokens.
- `千万俱乐部` — 10M+ tokens in a period.
- `赛博加班中` — heavy usage late at night.
- `额度边缘行走` — repeatedly ran near the 5h/weekly quota ceiling.

**Session behavior**
- `Context Marathoner` — a single very long / high-context session.
- `别问，问就是没 clear` — long sessions without a `/clear` (topic drift).
- `持之以恒` — used Claude Code on N consecutive days.
- `长对话爱好者` — high average turns per session.
- `一镜到底` — completed a big task in one unbroken session.

**Prompt style** (length/shape only — never semantic, unless opt-in + local)
- `小作文工程师` — consistently long prompts.
- `提示词卷王` — high prompt volume.
- `鲁迅式提示词` — terse-but-dense prompts.
- `一句话召唤术` — very short prompts that still got results.
- `极简 prompt 派` — minimal prompt style overall.

**Workflow / agentic coding**
- `全自动小公司` — heavy multi-agent / workflow usage.
- `Ship It` — many merged-style outcomes (local proxy: completed runs).
- `Hello World, Agent` — first sub-agent / workflow run.
- `自动接管 GitHub` / `Issue 召唤师` / `PR 炼金术士` / `测试也交给你了` — agentic
  coding milestones (issue handling, PR generation, test generation), gated on
  local workflow signals.

**Model behavior**
- `Sonnet 常驻民` / `Opus 重炮手` / `Haiku 轻骑兵` — dominant model family.
- `模型雨露均沾` — balanced across models.
- `某一家死忠粉` — near-exclusive use of one model.

**Plugin-specific**
- `设置全点亮` — explored most settings.
- `状态栏极简主义者` — minimal status-bar config.
- `优化器常客` — frequent Usage Optimizer use.
- `AI 建议听劝型用户` — acted on AI advice (local proxy).
- `仪表板考古学家` — viewed deep history.
- `每日看板打卡` — opened the dashboard daily.

**Hidden** (surprise unlocks, discovered not advertised)
- `Fable 临时体验卡` — used Fable.
- `古法编程` — a stretch with no agentic features.
- `超能蜘蛛` — (playful) very high parallel agent fan-out.
- `多核智人` — many concurrent sessions.
- `赛博奴隶主` — extreme agent delegation.
- `缓存炼金术` — exceptional cache efficiency.
- `凌晨三点的合伙人` — 3 a.m. coding.

## 4. Achievement levels

- **One-time** — unlocked once (first sub-agent, first Fable use).
- **Cumulative bronze / silver / gold** — tiered thresholds (e.g. tokens,
  consecutive days, sessions).
- **Hidden** — not listed until unlocked.
- **Seasonal / event** — time-boxed (a launch week, a month theme); local clock,
  no server.

## 5. Anti-gaming / anti-abuse

- Do **not** chase perfect fairness — this is for fun.
- Avoid high-stakes ranking; prefer an **opt-in showcase** over a leaderboard.
- Public data (if ever) is aggregated and anonymous; achievements are satisfying
  **locally** even with no public stats.
- No prompt-semantic classification unless clearly opt-in and local.

## 6. Community feature roadmap

- A GitHub Discussion to crowdsource achievement ideas.
- An optional public showcase (opt-in).
- A possible README badge export.
- "x% of users unlocked this" only **after** a sufficient opt-in sample — and no
  public ranking in the early stage.

## 7. Icon generation prompt template

Reusable template for generating badge art:

> Create a clean achievement badge icon for a developer productivity app.
> Achievement name: {name}
> Meaning: {description}
> Style: modern minimal, playful but not childish, rounded square badge, soft
> gradient, subtle shadow, high contrast, no text, suitable for 64×64 and
> 256×256.
> Visual metaphor: {metaphor}
> Avoid: copyrighted characters, overly complex details, photorealism, cluttered
> UI.

## 8. Implementation milestones

1. **V2.3 RC** — local-only achievements, 10–20 to start.
2. **V2.3 stable** — optional share-card integration (a badge row on the card).
3. **Later** — opt-in community showcase.
4. **Later** — GitHub profile / README export integration (see §9).

## 9. GitHub profile / community sharing (spec)

- **Approach:** a GitHub profile-README card (the same kind of usage/heatmap
  recap, rendered for a profile).
- The maintainer **manually tests** the profile-README workflow first (export an
  SVG / paste a Markdown snippet by hand) before any automation.
- **No OAuth in V2.2.** Future options, only after a privacy review:
  a. export an SVG manually;
  b. copy a Markdown snippet;
  c. an optional GitHub token / OAuth auto-update.
- Same privacy rules as the share card: aggregate only, opt-in, no prompts /
  paths / ids / usernames.
